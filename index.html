Top Issues

Here are some issues I frequently encounter with details on how to fix them:

Image missing alt attribute

Image alternatives are important for users using assistive technologies (like screenreaders). A description of the image helps users understand what information it's visually conveying. To do this set the "alt" attribute on all img tags. If the image is purely decorative, set the alt to be empty so it is ignored.

Example: informative image <img src="bird.jpg" alt="photo of a bluejay eating a worm">

Example: decorative image <img src="background.jpg" alt="">

Duplicate ID

IDs are used to identify unique elements within a webpage. They're useful to link to and reference, particularly for keyboard and assistive technology interactions. But if an ID is used more than once, these supports will break. Ensure IDs are only used once per page.

Example: div with id <div id="unique_id">

Blank link text

A link without link text will not be understandable to assistive technology users. This often happens for links that are font icons or images. Provide link text (that can be visually hidden) or an accessible label (aria-label attribute).

Example: link icon (that is displayed with CSS) <a href="http://mywebsite.org"><span class="sr-only">My website</span></a>

Language not set

Programmatically setting the language of your website will help assistive technologies correctly pronounce the content on your site.

Example: an English website <html lang="en">

Link text with identical title

Adding a title to link text could result in repetition and confusion for assistive technology users as both the title and link text may be read out. Simplify! Remove title attributes on links as they're not helpful.
